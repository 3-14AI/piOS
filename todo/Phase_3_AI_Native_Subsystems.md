4. Фаза III: Интеграция AI-Native Подсистем (AIOS) Цель фазы : Превратить ОС из пассивного исполнителя команд в интеллектуального агента. Внедрение LLM как "ядерного ресурса" (Kernel Resource), доступного через системные вызовы. Технологический стек : WASI-NN, Rust-based Inference (Burn/Candle), Vector DB. Раздел 3.1: Нейросетевой Интерфейс Ядра В традиционных ОС ИИ — это просто приложение. В AIOS — это часть инфраструктуры.

## WP-021 Реализация хоста WASI-NN

Описание : Реализация спецификации wasi-nn в ядре. Это мост между WASM- приложениями и тензорными вычислениями. Инсайт : Это стандартизирует доступ к "мозгу" системы. Любой драйвер или приложение может запросить инференс через унифицированный API, не заботясь о том, работает ли он на CPU, GPU или NPU.

## WP-022 Встроенное Векторное Хранилище (Kernel Vector Store)

Описание : Реализация легковесной векторной базы данных как системного сервиса (или привилегированного компонента). Назначение : Это "Долговременная Память" (Long Term Memory) операционной системы. Здесь хранятся логи, конфигурации, документация API и история действий пользователя в виде эмбеддингов.

## WP-023 Менеджер Контекстного Окна (Context Pager)

Описание : Подсистема, аналогичная менеджеру виртуальной памяти. Контекстное окно LLM ограничено (например, 128k токенов). Менеджер должен "странично" подгружать (page in/out) релевантную информацию из векторного хранилища в активный контекст модели в зависимости от текущей задачи. Инсайт : Контекст рассматривается как дефицитный ресурс, требующий планирования (Scheduling).

## WP-024 Драйвер VirtIO-GPU (2D/3D)

Описание : Поддержка графического ускорения. Критично не только для GUI, но и для ускорения инференса малых моделей, если NPU недоступен. Верификация : Проверка корректности очередей команд GPU для предотвращения зависаний (hangs) графической подсистемы.

## WP-025 Интеграция Inference Runtime (Candle/Mistral.rs)

Описание : Встраивание движка инференса (написанного на Rust, например, candle или mistral.rs) непосредственно в слой сервисов ядра. Оптимизация под no_std или минимальное окружение. Безопасность : Использование Rust гарантирует memory safety даже при сложной тензорной математике. ## WP-026 Системный вызов "Намерение" (sys_intent)

Описание : Новый тип syscall, принимающий не конкретные параметры (fd, buffer), а семантическое описание задачи (напр., "найти и прочитать конфиг сети"). Ядро использует встроенную SLM (Small Language Model) для трансляции интента в цепочку классических вызовов WASI.

## WP-028 Протокол Agent-to-Agent (A2A)

Описание : Разработка стандарта межпроцессного взаимодействия для ИИ-агентов внутри ОС. Позволяет "Агенту Планировщику" договариваться с "Агентом Сети" о приоритетах трафика. Инсайт : Это эволюция IPC для мира ИИ.

## WP-029 Интеграция Model Context Protocol (MCP)

Описание : Реализация стандарта MCP, позволяющего агентам ОС безопасно подключаться к внешним источникам данных и инструментам. Это делает ОС совместимой с растущей экосистемой ИИ-инструментов.

## WP-030 Слой Управления и Ограничений (Guardrails

Описание : Фильтр, перехватывающий действия Layer) ИИ. Верификация : Доказательство того, что никакой высокоуровневый интент ИИ не может нарушить низкоуровневые политики безопасности (ACL).
