# 4. Фаза III: Интеграция AI-Native Подсистем (AIOS)

**Цель фазы:** Превратить ОС из пассивного исполнителя команд в интеллектуального агента. Внедрение LLM как "ядерного ресурса" (Kernel Resource), доступного через системные вызовы.

**Технологический стек:** WASI-NN, Rust-based Inference (Burn/Candle), Vector DB.

## Роли и Риски

**Роль:** ИИ-интегратор (The AI Integrator).
*Специализация:* LLM, RAG, инференс-движки.

**Риск:** Задержки ИИ (Latency). Использование LLM для системных вызовов (см. WP-026) может внести неприемлемые задержки.
*Митигация:* Использовать иерархию моделей (SLM для быстрых решений, LLM для сложных), кэширование интентов и спекулятивное выполнение.

## Раздел 3.1: Нейросетевой Интерфейс Ядра
В традиционных ОС ИИ — это просто приложение. В AIOS — это часть инфраструктуры.

### WP-021 Реализация хоста WASI-NN
- [ ] **Реализовать спецификацию wasi-nn в ядре.**
  - Реализация моста между WASM-приложениями и тензорными вычислениями.
  - **Инсайт:** Стандартизация доступа к "мозгу" системы. Любой драйвер или приложение может запросить инференс через унифицированный API (CPU/GPU/NPU).

### WP-022 Встроенное Векторное Хранилище (Kernel Vector Store)
- [ ] **Реализовать легковесную векторную базу данных как системный сервис.**
  - **Назначение:** "Долговременная Память" (Long Term Memory) ОС.
  - Хранение логов, конфигураций, документации API и истории действий пользователя в виде эмбеддингов.

### WP-023 Менеджер Контекстного Окна (Context Pager)
- [ ] **Разработать подсистему управления контекстным окном LLM.**
  - Аналог менеджера виртуальной памяти для контекста (например, 128k токенов).
  - Реализация "страничной" подгрузки (page in/out) релевантной информации из векторного хранилища в активный контекст модели.
  - **Инсайт:** Контекст рассматривается как дефицитный ресурс, требующий планирования (Scheduling).

### WP-024 Драйвер VirtIO-GPU (2D/3D)
- [ ] **Реализовать поддержку графического ускорения.**
  - Критично для GUI и ускорения инференса малых моделей (если NPU недоступен).
  - **Верификация:** Проверка корректности очередей команд GPU для предотвращения зависаний (hangs) графической подсистемы.

### WP-025 Интеграция Inference Runtime (Candle/Mistral.rs)
- [ ] **Встроить движок инференса в слой сервисов ядра.**
  - Использовать Rust-based решение (например, candle или mistral.rs).
  - Оптимизация под no_std или минимальное окружение.
  - **Безопасность:** Использование Rust гарантирует memory safety даже при сложной тензорной математике.

## Раздел 3.2: Агентные Системные Вызовы
Создание протоколов, позволяющих программам общаться с ОС на естественном языке или через семантические интенты.

### WP-026 Системный вызов "Намерение" (sys_intent)
- [ ] **Реализовать новый тип syscall `sys_intent`.**
  - Принимает семантическое описание задачи вместо конкретных параметров (fd, buffer).
  - Интеграция встроенной SLM (Small Language Model) для трансляции интента в цепочку классических вызовов WASI.

### WP-027 Семантический Поиск Файлов
- [ ] **Заменить традиционный find/open на семантический поиск.**
  - Использование векторной БД (WP-022).
  - Индексация файловой системы нейросетью (поиск по содержимому/смыслу, а не только по имени).

### WP-028 Протокол Agent-to-Agent (A2A)
- [ ] **Разработать стандарт межпроцессного взаимодействия для ИИ-агентов.**
  - Позволяет агентам (например, Планировщику и Сети) договариваться о приоритетах.
  - **Инсайт:** Эволюция IPC для мира ИИ.

### WP-029 Интеграция Model Context Protocol (MCP)
- [ ] **Реализовать стандарт MCP.**
  - Позволяет агентам ОС безопасно подключаться к внешним источникам данных и инструментам.
  - Обеспечение совместимости с экосистемой ИИ-инструментов.

### WP-030 Слой Управления и Ограничений (Guardrails Layer)
- [ ] **Реализовать фильтр, перехватывающий действия ИИ.**
  - **Верификация:** Доказательство (Verus), что никакой высокоуровневый интент ИИ не может нарушить низкоуровневые политики безопасности (ACL).
  - Реализация "Reference Monitor", гарантирующего, что ИИ не сможет повредить ядро (даже ради "оптимизации").
