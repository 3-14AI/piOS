# 6. Фаза V: Пользовательский Опыт и Экосистема

**Цель фазы:** Сделать систему пригодной для использования человеком, предоставляя интерфейсы нового поколения.

**Технологический стек:** WGPU, Slint/Iced, Networking stack (Smoltcp).

## Роли

**Роль:** Системный архитектор (The Systems Architect).
*Специализация:* Rust, WASM, драйверы, низкоуровневое программирование.

## Раздел 5.1: Интерфейс и Сеть

- [ ] **WP-041: Сетевой Стек User-Space (Smoltcp)**
  - **Описание:** Портирование smoltcp (Rust network stack) в WASM-компонент.
  - **Верификация:** Проверка переходов состояний TCP (State Machine) на соответствие RFC.

- [ ] **WP-042: Композитор Окон (WGPU Compositor)**
  - **Описание:** Создание оконного менеджера на базе wgpu (WebGPU native). Обеспечивает аппаратное ускорение интерфейса. Безопасность памяти wgpu критична для стабильности GUI.

- [ ] **WP-043: Рендеринг Текста и Ввод**
  - **Описание:** Интеграция ab_glyph для шрифтов и обработки ввода. Связь ввода с семантическим поиском (WP-027).

- [ ] **WP-044: Натурально-языковая Оболочка (NL-SH)**
  - **Описание:** Замена bash. Командная строка, понимающая естественный язык ("Удали все временные файлы старше недели"). Преобразует текст в вызовы API ядра.

- [ ] **WP-045: Интеграция GUI Фреймворка (Slint)**
  - **Описание:** Портирование легковесного Rust GUI (Slint) для работы поверх композитора. Возможность "Генеративного UI", где интерфейс рисуется ИИ на лету под задачу.

## Раздел 5.2: Полировка и Безопасность

- [ ] **WP-046: Криптографический Сервис (wasi-crypto)**
  - **Описание:** Реализация криптографии. Верификация constant-time алгоритмов для защиты от атак по сторонним каналам (side-channel attacks).

- [ ] **WP-047: Secure Boot Chain Extension**
  - **Описание:** Расширение WP-001. Проверка цифровых подписей начальных WASM-компонентов перед загрузкой.

- [ ] **WP-048: Многопользовательская Система Прав**
  - **Описание:** Реализация UID/GID, но управляемая через систему Capabilities. ИИ-агент должен уважать эти границы.

- [ ] **WP-049: Системная Наблюдаемость (Observability)**
  - **Описание:** Внедрение трейсинга (OpenTelemetry) в ядро для визуализации "мыслей" ИИ и путей принятия решений планировщиком.

- [ ] **WP-050: Финальная стабилизация и Документация**
  - **Описание:** Стресс-тестирование, стабилизация доказательств Verus (устранение "хрупких" доказательств), написание документации для разработчиков драйверов.
