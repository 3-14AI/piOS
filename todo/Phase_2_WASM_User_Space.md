# Фаза 2: WebAssembly User Space и Модель Драйверов

**Цель:** Реализовать пространство пользователя, где вместо нативных бинарных файлов (ELF/PE) используется WebAssembly. Это основа для "Self-coding", обеспечивающая безопасную песочницу для кода, сгенерированного ИИ.

**Технологический стек:** Wasmtime (no_std fork), WASI, WIT.

## Роли и Риски

**Роль:** Системный архитектор (The Systems Architect).
*Специализация:* Rust, WASM, драйверы, низкоуровневое программирование.

**Риск:** Зрелость WASM Component Model. Стандарт все еще развивается.
*Митигация:* Активное участие в стандартах (Bytecode Alliance) и готовность писать собственные полифилы.

## Раздел 2.1: Интеграция WASM Runtime в Ядро

- [ ] **WP-011: Портирование WASM Runtime (no_std)**
  - **Описание:** Адаптация легковесного рантайма (wasmi/wasmtime) для работы без std. Замена аллокаций на ядерный GlobalAlloc.
  - **Инсайт:** Превращение ОС в "Unikernel" с изоляцией через WASM.

- [ ] **WP-012: Верификация песочницы памяти WASM**
  - **Описание:** Верификация "клея" между рантаймом и памятью ядра.
  - **Верификация:** Доказать в Verus, что `memory.read/write` никогда не выходят за пределы `WasmMemory`.

- [ ] **WP-013: Реализация WASI-Core (Syscalls)**
  - **Описание:** Реализация `wasi-io`, `wasi-filesystem` поверх IPC ядра.
  - **Верификация:** Проверка соответствия спецификации WASI и валидности прав доступа (Capabilities).

- [ ] **WP-014: Динамический Линковщик Компонентов (Linker)**
  - **Описание:** Механизм загрузки и связывания WASM-компонентов (Component Model). Подключение драйверов как плагинов.
  - **Важность:** Основа для горячей замены драйверов (Self-coding).

- [ ] **WP-015: Система Капабилити (Capabilities System)**
  - **Описание:** Реализация таблицы ресурсов (Resource Table). Доступ к железу через типизированные хендлы.
  - **Верификация:** Доказать невозможность подделки хендла (handle forging).

## Раздел 2.2: Инфраструктура Драйверов на WASM

- [ ] **WP-016: Драйвер VirtIO-Blk (WASM)**
  - **Описание:** Драйвер блочного устройства на Rust -> WASM.
  - **Верификация:** Доказательство корректности работы с Virtqueue (индексы avail/used в допустимых пределах).

- [ ] **WP-017: Драйвер VirtIO-Net (WASM)**
  - **Описание:** Сетевой драйвер.
  - **Верификация:** Доказательство отсутствия утечек памяти при передаче пакетов.

- [ ] **WP-018: PCI Bus Enumerator**
  - **Описание:** Сканирование шины PCI и загрузка WASM-драйверов.
  - **Верификация:** Корректность парсинга конфигурации и соответствие VendorID/DeviceID.

- [ ] **WP-019: Абстракция DMA (Safe DMA)**
  - **Описание:** Безопасный интерфейс для DMA (закрепленные буферы).
  - **Верификация:** Доказать, что регион DMA закреплен (pinned) во время I/O.

- [ ] **WP-020: Виртуальная Файловая Система (VFS)**
  - **Описание:** Объединение различных ФС.
  - **Верификация:** Отсутствие циклов в графе директорий и корректность блокировок (rw-locks).
