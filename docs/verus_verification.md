# Формальная Верификация с Verus

Одним из главных столпов надежности системы является использование инструмента `Verus` для формальной верификации кода.

## Ключевые аспекты верификации:

1.  **Гарантии Безопасности:**
    Verus используется для математического доказательства свойств программы, таких как отсутствие обращений к недействительной памяти, отсутствие состояний гонки (race conditions) при использовании примитивов синхронизации (например, Spinlock, TicketLock) и отсутствие переполнений (buffer overflows).

2.  **Структуры Данных и Аллокаторы:**
    Аллокатор физических страниц (PMM), таблица страниц (Paging) и ядерный кучевой аллокатор (Slab Allocator) формально верифицированы. В Verus используются токены (Ghost state) для доказательства уникального владения и корректного отображения виртуальной памяти в физическую.

3.  **Автоматическая Верификация и Z3:**
    Система интегрирует SMT-солвер (Z3) для верификации кода. Применяются сложные спецификации (pre-conditions, post-conditions, invariants), которые солвер разрешает на этапе компиляции.

4.  **Верификация переходов состояний:**
    Сложные конечные автоматы, такие как переходы состояний TCP-соединений в сетевом стеке (Smoltcp) и контроллерах прерываний (APIC/GIC), описаны и доказаны на уровне абстрактных моделей Verus.
