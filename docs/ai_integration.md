# AI-Native Интеграция

Система разработана с учетом глубокой интеграции искусственного интеллекта (ИИ) на всех уровнях.

## Ключевые подсистемы:

1.  **Inference Runtime и WASI-NN:**
    Слой инференса интегрирован напрямую через легковесную среду выполнения (Candle), адаптированную для no_std. Механизм `wasi-nn` обеспечивает мост между тензорными вычислениями и пользовательским пространством (WASM-компонентами).

2.  **Векторная База Данных (Vector DB):**
    Встроенная легковесная векторная БД хранит конфигурации, логи и историю выполнения в виде эмбеддингов. Она обеспечивает функционал семантического поиска в файловой системе (VFS).

3.  **LLM Контекст Менеджер:**
    Реализована подсистема управления контекстным окном (LLM Context Manager) как механизм "страничной памяти" для контекста агента. Автоматически подгружает и вытесняет (evicts) страницы на основе их релевантности и лимита токенов, применяя косинусное сходство через векторное хранилище.

4.  **Генерация Кода и Autopoiesis:**
    В ядре реализован пайплайн (Driver Generation Pipeline), способный к самостоятельному написанию драйверов, их верификации (Co-generation proofs), синтезу тестов и горячей перезагрузке. Ошибки верификации подаются обратно LLM для самокоррекции (Retry loop).

5.  **Agent-to-Agent (A2A) IPC:**
    Агенты общаются между собой посредством специального IPC-протокола, учитывающего приоритеты (AgentPriority), что позволяет управлять загрузкой системы и распределением задач. ИИ-действия ограничены через строго верифицированный слой Guardrails.
