# Архитектура Системы

Pi-OS (название условно) представляет собой операционную систему, ядро которой разработано с использованием языка Rust (в режиме `#![no_std]`) с упором на формальную верификацию.

## Основные компоненты архитектуры:
1.  **Микроядро (Microkernel):**
    Минималистичное ядро, предоставляющее только самые необходимые абстракции: управление потоками (threads), базовое планирование (scheduler) с поддержкой приоритетов и Round-Robin, механизмы межпроцессного взаимодействия (IPC), и управление физической памятью (PMM, виртуальная память).

2.  **Загрузка (Booting) и UEFI:**
    Система использует протоколы UEFI для перехода в Long Mode и начальной загрузки. Существует специальный загрузочный стаб (boot stub), подготавливающий карту памяти и передающий управление ядру с соблюдением формальных контрактов.

3.  **Виртуальная Файловая Система (VFS):**
    Управление файлами построено на базе графа директорий. Отличительной чертой является поддержка семантического поиска, реализованная благодаря векторному хранилищу.

4.  **Слой абстракций устройств:**
    Включает в себя драйверы VirtIO (VirtIO-Blk, VirtIO-Net, VirtIO-GPU) и абстракции прямого доступа к памяти (Safe DMA). Модель устройств базируется на изоляции, а драйверы могут загружаться динамически в виде WASM-компонентов.

5.  **Горячая Замена (Hot Reload) и Blue-Green развертывание:**
    Ядро поддерживает параллельное выполнение компонентов (shadow mode) и механизмы безопасного обновления состояния "на лету", обеспечивая высокую доступность.
